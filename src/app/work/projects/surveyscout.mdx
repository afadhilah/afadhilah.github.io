## Overview

**SurveyScout** adalah aplikasi mobile marketplace untuk survei lapangan yang dibangun menggunakan Flutter. Platform ini menghubungkan tiga aktor utama: **Client** yang membutuhkan data lapangan, **Surveyor** yang mengumpulkan data, dan **Respondent** yang mengisi survei online — semuanya dalam satu ekosistem yang terintegrasi.

Proyek ini dikembangkan sebagai bagian dari mata kuliah Rekayasa Perangkat Lunak di ITS Semester 7, dengan fokus pada arsitektur mobile yang scalable, role-based access control, dan desain UI/UX yang modern.

      code: `// router.dart — Role-based redirect
redirect: (context, state) {
  final authProvider = context.read<AuthProvider>();
  final isPublicRoute = publicRoutes.contains(state.matchedLocation);

  if (!authProvider.isAuthenticated && !isPublicRoute) {
    return '/login';
  }

  if (authProvider.isAuthenticated) {
    final role = authProvider.user?.role;
    if (role == 'client') return '/client/dashboard';
    if (role == 'surveyor') return '/surveyor/dashboard';
    if (role == 'respondent') return '/respondent/dashboard';
  }
  return null;
},`,
      language: "dart",
      label: "router.dart"
    },
    {
      code: `// database_helper.dart — SHA-256 auth
Future<Map<String, dynamic>?> loginUser(
  String email, String password
) async {
  final db = await database;
  final hashedPassword = _hashPassword(password);

  final result = await db.query(
    'users',
    where: 'email = ? AND password = ?',
    whereArgs: [email, hashedPassword],
  );

  if (result.isNotEmpty) {
    final token = _generateToken(email);
    final expiresAt = DateTime.now().add(Duration(days: 30));
    await db.insert('sessions', {
      'user_id': result.first['id'],
      'token': token,
      'expires_at': expiresAt.toIso8601String(),
    });
    return result.first;
  }
  return null;
}

String _hashPassword(String password) {
  final bytes = utf8.encode(password);
  return sha256.convert(bytes).toString();
}`,

## Database Schema

Menggunakan SQLite lokal dengan dua tabel utama dan indexing untuk performa query:

<CodeBlock
  marginBottom="16"
  codes={[
    {
      code: `-- users table
CREATE TABLE users (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  email      TEXT    UNIQUE NOT NULL,
  password   TEXT    NOT NULL,  -- SHA-256 hashed
  name       TEXT    NOT NULL,
  role       TEXT    NOT NULL,  -- client | surveyor | respondent | admin
  phone      TEXT,
  avatar_url TEXT,
  created_at TEXT    NOT NULL,  -- ISO 8601
  updated_at TEXT    NOT NULL
);

-- sessions table
CREATE TABLE sessions (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id    INTEGER NOT NULL REFERENCES users(id),
  token      TEXT    NOT NULL,  -- SHA-256 generated
  created_at TEXT    NOT NULL,
  expires_at TEXT    NOT NULL   -- 30 days TTL
);

-- Indexes untuk query performance
CREATE INDEX idx_users_email    ON users(email);
CREATE INDEX idx_sessions_token ON sessions(token);`,

## Outcome & Pembelajaran

Proyek ini memberikan pengalaman praktis membangun **full-stack mobile application** dengan kompleksitas nyata:

- Merancang **role-based access control** yang aman menggunakan declarative routing
- Mengimplementasikan **local-first architecture** dengan SQLite dan sinkronisasi state via Provider
- Membangun **design system yang konsisten** untuk tim yang mengerjakan banyak screen secara paralel
- Memahami trade-off antara **mock API** untuk development velocity vs live backend untuk production readiness
- Mengelola **state yang kompleks** (auth, navigation, data) dalam satu aplikasi multi-role

SurveyScout membuktikan bahwa Flutter mampu menangani aplikasi dengan skala menengah-besar dengan arsitektur yang clean, performa yang baik, dan developer experience yang solid.
